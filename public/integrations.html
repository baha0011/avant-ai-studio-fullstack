Backend та інтеграції

Заявка проходить повний шлях: сайт → API → база даних → Google Sheets → Telegram.

Це вже не статичний сайт. Проект має Express backend, SQLite DB, Google Sheets API, Telegram Bot API та внутрішню admin-панель.

POST /api/leads
GET  /api/leads
PATCH /api/leads/:id/status
GET  /api/health
FrontendФорма на сайті збирає заявку клієнта.
Backend APIExpress перевіряє дані та створює заявку.
DatabaseSQLite зберігає заявку та статуси інтеграцій.
Google Sheets + TelegramДані йдуть у таблицю та адміну в Telegram.
База данихSQLite
Таблиця leads містить ID заявки, ім’я, контакт, нішу, опис задачі, мову, статус, статус Google Sheets, статус Telegram і дату створення.
Google Sheets замість CRMGoogle API
Google Sheets використовується як простий операційний центр для команди. Таблиця отримує новий рядок після кожної заявки.
Telegram для адміністратораBot API
Адмін отримує красиве структуроване повідомлення в особистий чат, групу або канал. Це швидко, зручно і не потребує складної CRM.
Що потрібно налаштувати.env
У файлі .env потрібно вказати токен Telegram-бота, chat_id, ID Google Sheet, email сервісного акаунта та приватний ключ.
